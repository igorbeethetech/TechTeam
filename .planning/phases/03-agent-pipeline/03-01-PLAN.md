       - Import `query` from `@anthropic-ai/claude-agent-sdk`

       - Export `executeAgent(params: { prompt: string, schema: object, timeoutMs: number, cwd?: string })` that returns `Promise<{ output: unknown, tokensIn: number, tokensOut: number, costUsd: number, durationMs: number }>`
       - Implementation: Create AbortController + setTimeout for timeout. Iterate over `query({ prompt, options: { abortController, model: config.CLAUDE_MODEL, maxTurns: 5, outputFormat: { type: 'json_schema', schema }, tools: [], permissionMode: 'bypassPermissions' } })`. Collect the `result` message. clearTimeout in finally block. Check `result.subtype === 'success'`, throw on failure. Return `{ output: result.structured_output, tokensIn: result.usage.input_tokens, tokensOut: result.usage.output_tokens, costUsd: result.total_cost_usd, durationMs: result.duration_ms }`.
       - Wrap AbortError specifically: if `error.name === 'AbortError'`, throw new Error with message "Agent execution timed out after {timeoutMs}ms".

    2. Create `apps/api/src/queues/agent.worker.ts` -- worker processor:
       - Import `Worker`, `Job` from `bullmq`
       - Import `createWorkerConnection` from `../lib/redis.js`
       - Import `forTenant` from `@techteam/database`
       - Import `AgentJobData`, `AgentJobResult`, `agentQueue` from `./agent.queue.js`
       - Define `PHASE_TIMEOUTS: Record<string, number>` = `{ discovery: 2 * 60 * 1000, planning: 5 * 60 * 1000 }` (AGENT-07 requirement)
       - Export `createAgentWorker()` function that:
         a. Creates `Worker<AgentJobData, AgentJobResult>('agent-pipeline', processor, { connection: createWorkerConnection(), concurrency: 2 })`
         b. Processor function: gets job data, creates tenant-scoped prisma via `forTenant(tenantId)`, creates AgentRun record with status 'running' and attempt number from `job.attemptsMade + 1`, dynamically imports the correct agent function based on `phase` (lazy import to avoid circular deps: `const { runDiscoveryAgent } = await import('../agents/discovery.agent.js')` or `const { runPlanningAgent } = await import('../agents/planning.agent.js')`), calls agent, updates AgentRun to 'completed' with results, updates Demand with output (requirements or plan field), updates Demand totalTokens/totalCostUsd via increment, handles stage advancement:
            - If discovery + no ambiguities: set stage to 'planning', set agentStatus to null, enqueue planning job
            - If discovery + has ambiguities: keep stage 'discovery', set agentStatus to 'paused'
            - If planning: set stage to 'development', set agentStatus to null (parked for Phase 4)
         c. On error in processor: update AgentRun to 'failed' (or 'timeout' if AbortError) with error message, set Demand agentStatus to 'failed'. Re-throw so BullMQ handles retry.
         d. Attach `worker.on('error', (err) => console.error('Worker error:', err))` -- CRITICAL per BullMQ docs
         e. Return worker instance for graceful shutdown
