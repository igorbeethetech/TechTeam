**1. Prisma Schema -- Add Demand model and enums:**

Add to `packages/database/prisma/schema.prisma` after the Project model:

```prisma
model Demand {
  id              String         @id @default(cuid())
  tenantId        String
  projectId       String
  title           String
  description     String?        @db.Text
  stage           DemandStage    @default(inbox)
  priority        DemandPriority @default(medium)
  complexity      Complexity?
  requirements    Json?
  plan            Json?
  branchName      String?
  prUrl           String?
  mergeStatus     MergeStatus?
  mergeConflicts  Json?
  mergeAttempts   Int            @default(0)
  totalTokens     Int            @default(0)
  totalCostUsd    Float          @default(0)
  createdBy       String
  createdAt       DateTime       @default(now())
  updatedAt       DateTime       @updatedAt

  project         Project        @relation(fields: [projectId], references: [id], onDelete: Cascade)

  @@index([tenantId])
  @@index([projectId])
  @@index([stage])
  @@index([projectId, stage])
}

  inbox
  discovery
  planning
  development
  testing
  merge
  done
```

**3. Shared schemas and types:**

Create `packages/shared/src/schemas/demand.ts`:
```typescript
import { z } from "zod"

export const demandCreateSchema = z.object({
  projectId: z.string().min(1, "Project is required"),
  title: z.string().min(2, "Title must be at least 2 characters").max(200),
  description: z.string().max(5000).optional(),
  priority: z.enum(["low", "medium", "high", "urgent"]).default("medium"),
})

export const demandUpdateSchema = demandCreateSchema.partial().omit({ projectId: true })

export const demandStageUpdateSchema = z.object({
  stage: z.enum(["inbox", "discovery", "planning", "development", "testing", "merge", "done"]),
})

export type DemandCreate = z.infer<typeof demandCreateSchema>
export type DemandUpdate = z.infer<typeof demandUpdateSchema>
export type DemandStageUpdate = z.infer<typeof demandStageUpdateSchema>
```

Add to `packages/shared/src/types/index.ts`:
```typescript
export type DemandStage = "inbox" | "discovery" | "planning" | "development" | "testing" | "merge" | "done"
export type DemandPriority = "low" | "medium" | "high" | "urgent"
```

Add to `packages/shared/src/constants/index.ts`:
```typescript
export const PRIORITY_LEVELS = ["low", "medium", "high", "urgent"] as const
export type PriorityLevel = (typeof PRIORITY_LEVELS)[number]
```

Update `packages/shared/src/index.ts` to export all new demand schemas, types, and constants.

**4. Demand API routes:**

Create `apps/api/src/routes/demands.ts` following the exact same pattern as `projects.ts`:

- `GET /` -- List demands. Accept optional `projectId` query param. If `projectId` provided, filter by it. Order by `createdAt asc`. Return `{ demands }`.
- `POST /` -- Create demand. Parse body with `demandCreateSchema`. Set `createdBy` from `request.session.user.id`. Return 201 with `{ demand }`.
- `GET /:id` -- Get single demand by ID. Return 404 if not found. Return `{ demand }`.
- `PATCH /:id/stage` -- Update demand stage (used by drag-and-drop). Parse body with `demandStageUpdateSchema`. Return `{ demand }`.

Use `request.prisma.demand` for all queries (tenant-scoped automatically).
