- Stage labels: Inbox, Discovery, Planning, Development, Testing, Merge, Done

**3. Demand detail component:**

Create `apps/web/src/components/demands/demand-detail.tsx`:
- Accept a `demand` object prop with all Demand fields
- Layout:
  - Back button (arrow-left icon + "Back to Board" or "Back" text) linking to `/projects/${demand.projectId}/board`
  - Title as h1
  - Pipeline progress bar showing current stage
  - Metadata section: Priority badge (same color coding as demand card), created date, created by (if available)
  - Description section (if description exists) -- render as prose text
  - Cost section: Total tokens and total cost (if > 0)
  - Stage info section: Current stage name prominently displayed
- Future-phase fields (complexity, requirements, plan, branchName, prUrl, mergeStatus) should be structured for later display but only shown when not null -- this provides progressive disclosure as demands move through stages

**4. Demand detail page:**

Create `apps/web/src/app/(dashboard)/demands/[demandId]/page.tsx`:

- Loading state with skeleton
- Error/not-found state
- Render DemandDetail component with fetched demand data
