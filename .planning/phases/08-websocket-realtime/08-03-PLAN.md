1. Create `apps/web/src/hooks/use-websocket.ts`:

   **Imports:** `useEffect`, `useRef`, `useCallback`, `useState`, `createContext`, `useContext`, `type ReactNode` from react; `useQueryClient` from `@tanstack/react-query`

   **Types:**
   ```typescript
   type ConnectionStatus = "connecting" | "connected" | "disconnected"
   ```

   **Event-to-query-key mapping** (this is the core invalidation logic):
   ```typescript
   const EVENT_TO_QUERY_KEYS: Record<string, (payload: any) => string[][]> = {
     "demand:updated": (p) => [["demand", p.demandId], ["demands", p.projectId]],
     "demand:stage-changed": (p) => [["demands", p.projectId]],
     "agent:status-changed": (p) => [["demand", p.demandId], ["demands", p.projectId]],
     "agent-run:updated": (p) => [["agent-runs", p.demandId]],
     "notification:created": () => [["notifications", "unread-count"], ["notifications"]],
   }
   ```

   **WebSocket context:**
   - Create `WebSocketContext = createContext<ConnectionStatus>("disconnected")`
   - Export `useWsStatus()` hook that returns `useContext(WebSocketContext)` -- this is what components use to check connection status

   **WebSocketProvider component:**
   - This is a `"use client"` component that wraps children
   - Uses `useQueryClient()` to get the query client
   - Manages a single WebSocket connection for the entire app
   - Exposes status via context provider

   **Connection logic inside WebSocketProvider:**
   - Build WS URL: `const wsUrl = \`${(process.env.NEXT_PUBLIC_API_URL || "http://localhost:3010").replace("http", "ws")}/ws\``
   - Create WebSocket with native `new WebSocket(wsUrl)` -- browser sends cookies automatically
   - `ws.onopen`: set status "connected", reset backoff to 1000ms
   - `ws.onmessage`: parse JSON, look up `EVENT_TO_QUERY_KEYS[data.type]`, call `queryClient.invalidateQueries({ queryKey })` for each matched key. Wrap in try/catch (malformed messages should not crash the app).
   - `ws.onclose`: set status "disconnected", schedule reconnect with exponential backoff (start 1s, max 30s, add random jitter 0-1s)
   - `ws.onerror`: let onclose handle reconnection (onerror always fires before onclose)
   - Cleanup on unmount: clear reconnect timer, close WebSocket

   **Reconnection backoff:**
   - Use `useRef` for: wsRef (WebSocket instance), reconnectTimerRef (setTimeout id), reconnectDelayRef (current delay, starts at 1000)
   - On close: `const jitter = Math.random() * 1000; setTimeout(connect, reconnectDelayRef.current + jitter); reconnectDelayRef.current = Math.min(reconnectDelayRef.current * 2, 30000)`
   - On open: `reconnectDelayRef.current = 1000` (reset)

   **Export:** `WebSocketProvider` (component) and `useWsStatus` (hook)

2. Update `apps/web/src/components/providers.tsx`:
   - Import `WebSocketProvider` from `@/hooks/use-websocket`
   - Wrap children: `<QueryClientProvider client={queryClient}><WebSocketProvider>{children}</WebSocketProvider></QueryClientProvider>`
   - WebSocketProvider MUST be inside QueryClientProvider (it calls useQueryClient)

   **IMPORTANT:** The `"use client"` directive is already present in providers.tsx. WebSocketProvider also needs `"use client"` in its own file since it uses hooks.
