2. Create `apps/api/src/plugins/websocket.ts`:
   - Import `fp` from `fastify-plugin` and `websocket` from `@fastify/websocket`
   - Register `@fastify/websocket` on the Fastify instance
   - Export via `fp(websocketPlugin, { name: "websocket-plugin" })`

3. Create `apps/api/src/routes/ws.ts` -- this is the core WebSocket handler:
   - Import types: `FastifyInstance`, `WebSocket` from `ws`, `WsEvent` from `@techteam/shared`
   - Import `createSubscriberConnection` from `../lib/redis.js`

   **Tenant connection map:**
   - `const tenantConnections = new Map<string, Set<WebSocket>>()`

   **Redis PubSub subscriber:**
   - Create a dedicated subscriber connection (lazy, created on first client connect)
   - `subscriber.on("message", (channel, message) => { ... })` -- parse the message, extract tenantId from channel (`ws:tenant:{tenantId}`), call `broadcastToTenant(tenantId, message)`
   - Dynamic subscription: subscribe to `ws:tenant:{tenantId}` when first client for that tenant connects, unsubscribe when last client disconnects

   **broadcastToTenant function:**
   - `export function broadcastToTenant(tenantId: string, message: string): void`
   - Iterate `tenantConnections.get(tenantId)`, send to each socket where `readyState === WebSocket.OPEN`

   **WebSocket route:**
   - Register `GET /ws` with `{ websocket: true }`
   - The route is registered INSIDE the protected scope (same as other routes), so the auth plugin's preHandler hook fires automatically
   - In the handler `(socket, request)`:
     - Extract `tenantId = request.session?.session?.activeOrganizationId`
     - If no tenantId, close socket with code 4001 "No tenant context" and return
     - Add socket to `tenantConnections` for this tenantId
     - Subscribe to Redis channel if first client for this tenant
     - On `socket.close`: remove from set, unsubscribe from Redis if last client, delete tenant entry if empty
     - On `socket.error`: same cleanup as close

   **Heartbeat (prevent zombie connections):**
   - Set a 30-second interval that pings all connected sockets
   - Track `isAlive` flag per socket: set true on `pong`, set false before ping
   - If `isAlive` is false when ping fires, terminate the connection (zombie)
   - Clear the heartbeat interval when the Fastify server closes (`fastify.addHook("onClose", ...)`)

   **IMPORTANT auth note:** The existing auth plugin uses `preHandler` hook (not `preValidation`). @fastify/websocket routes respect Fastify hooks. The browser sends cookies on WebSocket upgrade requests automatically (same localhost origin). If Better Auth's getSession fails on the upgrade request, add a fallback: manually extract the session cookie from `request.headers.cookie`, build a Headers object, and call `auth.api.getSession({ headers })` inside the handler. But try the hook approach first -- it should work since the upgrade is still an HTTP request with cookies.

4. Update `apps/api/src/server.ts`:

   - Inside the existing `protectedApp` scope (where authPlugin + tenantPlugin are registered), add: `await protectedApp.register(wsRoutes)`
   - NOTE: Do NOT add a prefix to wsRoutes -- the route itself defines `/ws`
   - NOTE: The ws route will go through authPlugin preHandler but we need to handle the case where tenantPlugin's 403 reply might interfere with WS upgrade. Since tenantPlugin also fires preHandler, and it will attach `request.prisma`, this should be fine. The ws handler doesn't use request.prisma -- it only reads request.session for tenantId. If tenantPlugin returns 403 for missing org, that's correct behavior (no tenant = no WS).
