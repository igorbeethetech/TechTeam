<objective>
Add Redis PubSub event emission to all worker state mutation points, so every demand change, agent status update, and notification triggers a WebSocket event that the API server can forward to connected clients.

Purpose: Workers are the source of truth for pipeline state changes. Without event emission, the WebSocket server has nothing to broadcast -- this plan closes the "event source" side of the real-time pipeline.
Output: agent.worker.ts and merge.worker.ts publish WsEvents to Redis PubSub after every relevant Prisma mutation.
</objective>

IMPORTANT: Every `publishWsEvent` call is already fire-and-forget (the function internally catches errors). But since we're `await`-ing it, the await is just for backpressure -- if it somehow throws despite the try/catch, wrap each call in its own try/catch to be safe. ```typescript

```typescript
}
```

Then use `await emitEvent(...)` everywhere instead of `await publishWsEvent(...)` for double safety.
