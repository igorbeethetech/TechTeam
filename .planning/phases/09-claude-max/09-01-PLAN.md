**File 1: `apps/api/src/agents/base-agent-cli.ts`**

Create a CLI executor function `executeAgentCli` that:
- Accepts the same `AgentExecutionParams` interface from `base-agent.ts`
- Spawns `claude` with args: `--output-format json`, `--max-turns N`, `--no-session-persistence`
- If `params.schema` exists, add `--json-schema` with JSON.stringify of the schema
- If `params.systemPrompt` exists, add `--system-prompt`
- If `params.allowedTools` has items, add each tool as a separate arg after `--allowedTools`, plus `--dangerously-skip-permissions`. If no tools, pass nothing (CLI defaults work fine).
- If `params.model` exists, add `--model`
- Use `spawn` from `node:child_process` with `shell: process.platform === "win32"` and `env: { ...process.env }`
- Use `stdio: ["pipe", "pipe", "pipe"]` -- stdin is pipe, NOT ignore. This is critical.
- Write `params.prompt` to `child.stdin` and close it (stdin piping avoids command-line length limits for development agent's long prompts). Do NOT pass prompt via `-p` arg. Instead, pipe it via stdin. The CLI reads from stdin when no `-p` argument is provided.
- Collect stdout and stderr via streaming (data events, concatenate chunks)
- Implement timeout via `setTimeout` + `child.kill("SIGTERM")`
- On `child.on("error")`: check for ENOENT and provide a clear error message ("Claude CLI not found. Ensure 'claude' is installed globally and available in PATH.")
- On `child.on("close")`: parse stdout as JSON. If `json.is_error === true`, reject with the error result. Otherwise, return `AgentExecutionResult` with:
  - `output: json.structured_output ?? json.result`
  - `tokensIn: 0` (CLI does not expose token counts)
  - `tokensOut: 0`
  - `costUsd: json.total_cost_usd ?? 0`
  - `durationMs: json.duration_ms ?? (Date.now() - startTime)`
- If JSON parse fails, reject with clear error including first 500 chars of raw stdout

Import types from `./base-agent.js`:
```typescript
import type { AgentExecutionParams, AgentExecutionResult } from "./base-agent.js"
```

**File 2: `apps/api/src/agents/agent-router.ts`**

Create a routing function `executeAgentAuto` that:
- Accepts `tenantId: string` as first param and `params: AgentExecutionParams` as second
- Imports `executeAgent` from `./base-agent.js` and `executeAgentCli` from `./base-agent-cli.js`
- Imports `prisma` from `@techteam/database` (the raw, non-tenant-scoped client -- TenantSettings is a global lookup by tenantId)
- Looks up `TenantSettings` by tenantId: `prisma.tenantSettings.findUnique({ where: { tenantId } })`
- Reads `settings?.agentExecutionMode ?? "sdk"` (default to sdk if no settings record)
- If mode is `"cli"`, call `executeAgentCli(params)` and return the result
- If mode is `"sdk"` (or default), call `executeAgent(params)` and return the result
- Export both the function and re-export the types: `export type { AgentExecutionParams, AgentExecutionResult } from "./base-agent.js"`

This router is the ONLY new dependency agents will need in Plan 02. Agents will change one import line: `executeAgent` -> `executeAgentAuto` from `agent-router.js` instead of `base-agent.js`.
