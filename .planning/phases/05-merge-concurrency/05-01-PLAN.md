    - Queue name: `"merge-queue"`
    - Connection: `createWorkerConnection()`
    - Concurrency: `1` (serialize all merges globally -- FIFO)

    The worker callback `processMergeJob(job: Job<MergeJobData>)`:
    ```
    1. Extract demandId, tenantId, projectId from job.data
    2. Get tenant-scoped prisma via forTenant(tenantId)
    3. Load demand (findUniqueOrThrow) and project (findUniqueOrThrow)
    4. Guard: if demand.branchName is null, throw error
    5. Guard: if demand.prUrl is null, throw error
    6. Update demand: mergeStatus = "pending", mergeAttempts increment by 1, agentStatus = "running"
    7. Log: "[merge-worker] Processing merge: demandId=${demandId}, branch=${branchName}"

    // ---- STEP 1: Auto-merge ----
    8. Dynamic import { mergeFromBranch, resetWorkingDir, checkConflictMarkers } from "../lib/git.js"
    9. Dynamic import { closePullRequest, extractPrNumber } from "../lib/github.js"
    10. Call mergeFromBranch(project.repoPath, demand.branchName, project.defaultBranch)
    11. If merge.success:
        a. Optionally run post-merge tests: dynamic import child_process, spawn `npm test` in project.repoPath with 5-min timeout. Capture exit code. If tests fail, reset with git.reset(["--hard", `origin/${project.defaultBranch}`]) and throw Error("Post-merge tests failed").
           NOTE: For v1, make the test step lenient -- wrap in try/catch. If no test script exists or tests fail, log a warning but continue with merge. The testing agent already validated the code before merge.
        b. Push: `createGitClient(project.repoPath).push("origin", project.defaultBranch)`
        c. Close PR: `closePullRequest({ repoUrl: project.repoUrl, prNumber: extractPrNumber(demand.prUrl) })`
        d. Update demand: `mergeStatus = "merged", stage = "done", agentStatus = null`
        e. Log success and return `{ merged: true, mergeStatus: "merged" }`
    12. If !merge.success (conflicts detected):
        a. Log conflict files
        b. STEP 2 STUB: For now, directly fall through to Step 3
        c. STEP 3 STUB: Update demand: `mergeStatus = "needs_human", mergeConflicts = { files: merge.conflictFiles }, agentStatus = null`
        d. Return `{ merged: false, mergeStatus: "needs_human", error: "Merge conflicts detected" }`
    ```

    Add error handler: `worker.on("error", ...)` -- same pattern as agent worker.

    On job failure (catch block):
    - Update demand: `mergeStatus = "needs_human", agentStatus = null`
    - `resetWorkingDir(project.repoPath)` in finally block to clean up
    - Re-throw error for BullMQ retry
