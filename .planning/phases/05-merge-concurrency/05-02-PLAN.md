    Follow the pure agent function pattern from discovery/planning/development agents:

    ```typescript
    interface MergeResolverParams {
      demandId: string
      tenantId: string
      projectId: string
      repoPath: string
      branchName: string
      defaultBranch: string
      conflictFiles: string[]
      timeout: number
    }

    interface MergeResolverResult {
      resolved: boolean
      output: unknown
      tokensIn: number
      tokensOut: number
      costUsd: number
      durationMs: number
    }
    ```

    - Import `executeAgent` from `../agents/base-agent.js` and `mergeResolverOutputSchema` from `@techteam/shared`
    - Import `zodToJsonSchema` from `zod-to-json-schema` (already installed)
    - Implement `runMergeResolverAgent(params: MergeResolverParams): Promise<MergeResolverResult>`:
      a. Build a prompt that describes the merge conflict context:
         - "You are resolving merge conflicts in a repository."
         - "The branch `{branchName}` is being merged into `{defaultBranch}`."
         - "The following files have conflicts: {conflictFiles.join(', ')}"
         - "Your task: 1) Read each conflicted file. 2) Understand both sides of the conflict (the content between <<<<<<< HEAD and =======, and between ======= and >>>>>>> {branchName}). 3) Resolve each conflict by choosing the correct resolution that preserves both changes when possible. 4) Write the resolved file content (removing ALL conflict markers). 5) After resolving all files, run `git add .` to stage the resolutions. 6) Do NOT run git commit, git merge, or git push."
         - "IMPORTANT: You must remove ALL conflict markers (<<<<<<< ======= >>>>>>>). Partially resolved files are worse than unresolved ones."
      b. Call `executeAgent` with:
         - `prompt`: the built prompt
         - `schema`: `zodToJsonSchema(mergeResolverOutputSchema)`
         - `timeoutMs`: `params.timeout`
         - `cwd`: `params.repoPath`
         - `allowedTools`: `["Read", "Write", "Edit", "Bash", "Glob", "Grep"]` (same as development agent -- needs Write/Edit to resolve conflicts)
         - `maxTurns`: 30 (conflict resolution is bounded)
         - `systemPrompt`: "You are an expert merge conflict resolver. You understand code semantics and can merge changes from two branches intelligently. Your goal is to produce clean, working code that incorporates changes from both sides. When in doubt, prefer the feature branch changes but preserve all functionality."
         - `model`: `config.CLAUDE_DEV_MODEL` (use the development model for code understanding)
      c. Parse the output with `mergeResolverOutputSchema.safeParse(result.output)`. If parsing fails, return `{ resolved: false, output: null, ...metrics }`.
      d. Return `{ resolved: parsed.data.resolved && parsed.data.unresolvedFiles.length === 0, output: parsed.data, ...metrics }`
    - Export `runMergeResolverAgent`

    **2. Update `apps/api/src/queues/merge.worker.ts`:**

    Replace the Step 2/3 stubs with full implementation:

    After Step 1 fails with conflicts (the `!merge.success` branch):

    **Step 2: AI Conflict Resolution:**
    a. Update demand: `mergeStatus = "conflict_resolving"`
    b. Create an AgentRun record for the merge-resolver: `prisma.agentRun.create({ data: { demandId, phase: "merge", status: "running", attempt: demand.mergeAttempts + 1 } })`
    c. Re-attempt the merge to leave conflict markers in the working directory:
       ```typescript
       const git = createGitClient(project.repoPath)
       await git.fetch("origin")
       await git.checkout(project.defaultBranch)
       await git.pull("origin", project.defaultBranch)
       try { await git.merge([demand.branchName!]) } catch { /* expected -- conflicts */ }
       ```
    d. Get conflicted files: `const status = await git.status(); const conflictedFiles = status.conflicted`
    e. If `conflictedFiles.length > 0`:
       - Dynamic import `runMergeResolverAgent` from `../agents/merge-resolver.agent.js`
       - Call it with `{ demandId, tenantId, projectId, repoPath: project.repoPath, branchName: demand.branchName, defaultBranch: project.defaultBranch, conflictFiles: conflictedFiles, timeout: 10 * 60 * 1000 }`
       - Update the AgentRun with result metrics (tokensIn, tokensOut, costUsd, durationMs, status, output)
       - Accumulate costs on demand (same pattern as other agents)
       - Check resolution: `const postStatus = await git.status(); const stillConflicted = postStatus.conflicted.length > 0`
       - Also check for lingering conflict markers: `const hasMarkers = await checkConflictMarkers(project.repoPath)`
       - If `agentResult.resolved && !stillConflicted && !hasMarkers`:
         * Commit: `await git.commit(\`merge: AI-resolved conflicts for demand/${demandId}\`)`
         * Push: `await git.push("origin", project.defaultBranch)`
         * Close PR: `await closePullRequest({ repoUrl: project.repoUrl, prNumber: extractPrNumber(demand.prUrl!) })`
         * Update demand: `mergeStatus = "merged", stage = "done", agentStatus = null`
         * Log success and return `{ merged: true, mergeStatus: "merged" }`
       - Else (AI resolution failed):
         * `await git.merge(["--abort"]).catch(() => {})` -- may need reset instead
         * `await git.reset(["--hard", \`origin/${project.defaultBranch}\`])` -- clean reset
         * Fall through to Step 3

    **Step 3: Escalate to Human:**
    f. Update demand:
       ```typescript
       mergeStatus = "needs_human"
       mergeConflicts = { files: merge.conflictFiles, attemptedAI: true, mergeAttempts: demand.mergeAttempts + 1 }
       agentStatus = null
       ```
    g. Log: `"[merge-worker] Merge escalated to human: demandId=${demandId}, conflicts in ${merge.conflictFiles.length} files"`
    h. Return `{ merged: false, mergeStatus: "needs_human" }`

    NOTE: Keep the error handler from Plan 01 that resets working dir on failure.
