**Metrics API** in `apps/api/src/routes/metrics.ts` (new file):

Create a Fastify route plugin with prefix `/api/metrics` containing 4 GET endpoints. All endpoints use `request.prisma` (tenant-scoped) for Prisma Client operations, and import `prisma` from `@techteam/database` for `$queryRaw` (since raw SQL bypasses the tenant extension -- research pitfall 2). For raw queries, always include `WHERE "tenantId" = ${tenantId}` with `tenantId` from `request.session!.session.activeOrganizationId!`.

**GET /cost** (METR-01 -- cost per project, current month):
- Calculate `startOfMonth` as first day of current month.
- Use `request.prisma.demand.groupBy({ by: ["projectId"], _sum: { totalCostUsd: true }, _count: { id: true }, where: { createdAt: { gte: startOfMonth } } })`.
- Join with project names via `request.prisma.project.findMany({ where: { id: { in: projectIds } }, select: { id: true, name: true } })`.
- Return `{ costs: [{ projectId, projectName, totalCostUsd, demandCount }] }`.

**GET /throughput** (METR-02 -- demands completed per week, last 12 weeks):
- Use `prisma.$queryRaw` (NOT request.prisma -- raw SQL bypasses extension).
- Query:
```sql
SELECT
  EXTRACT(ISOYEAR FROM "completedAt")::int AS year,
  EXTRACT(WEEK FROM "completedAt")::int AS week,
  COUNT(*)::int AS count
FROM "Demand"
WHERE "tenantId" = ${tenantId}
  AND "stage" = 'done'
  AND "completedAt" IS NOT NULL
  AND "completedAt" >= NOW() - INTERVAL '12 weeks'
GROUP BY year, week
ORDER BY year, week
```
- CRITICAL: Cast `COUNT(*)::int` (not bigint) to avoid JSON serialization errors (research pitfall 1).
- Return `{ throughput: [{ year, week, count, label }] }` where `label` is formatted as `"W{week}"`.

**GET /avg-time-per-phase** (METR-03 -- average agent duration per phase):
- Use `request.prisma.agentRun.groupBy({ by: ["phase"], _avg: { durationMs: true }, _count: { id: true }, where: { status: "completed" } })`.
- Return `{ phases: [{ phase, avgDurationMs, totalRuns }] }`.

**GET /agent-success-rate** (METR-04 -- agent success rate):
- Use `request.prisma.agentRun.groupBy({ by: ["status"], _count: { id: true } })`.
- Calculate total, completed, failed from grouped results.
- Return `{ total, completed, failed, successRate (0-100), byStatus: [{ status, count }] }`.
