**Navigation items array** (top section):
- Dashboard: href="/", icon=LayoutDashboard
- Boards: href="/boards", icon=Kanban
- Projects: href="/projects", icon=FolderOpen
- Metrics: href="/metrics", icon=BarChart3
- Settings: href="/settings", icon=Settings

**Active state detection:**
- Dashboard ("/") is active when `pathname === "/"`
- Others: `pathname === href || pathname.startsWith(href + "/")`

**Project list section:**
- Fetch projects using `useQuery({ queryKey: ["projects"], queryFn: () => api.get<{ projects: { id: string; name: string }[] }>("/api/projects") })` -- uses the SAME query key as the Projects page so cache is shared and creating/archiving a project invalidates both.
- Each project renders as a SidebarMenuItem with SidebarMenuButton linking to `/projects/${project.id}/board`.
- Active when `pathname === /projects/${project.id}/board`.
- Use the Kanban icon for each project item.
- Show `<SidebarMenuSkeleton />` (3 items) while projects are loading.
- If no projects found (empty array), show a subtle "No projects yet" text.

**Footer section (Jira/Slack style):**
- User email displayed with truncation (truncate class).
- Logout button with LogOut icon.
- Both use SidebarMenuButton with tooltip props for collapsed state.
- Logout calls `signOut()` then `router.push("/login")`.

**Mobile sidebar close on navigation:**
- Get `setOpenMobile` from `useSidebar()` hook.
- On every nav link click, call `setOpenMobile(false)` to close the mobile sheet overlay.
- Apply this to both nav items and project items.

**Tooltip on collapsed sidebar:**
- Set `tooltip={label}` on every SidebarMenuButton (for nav items) and `tooltip={project.name}` (for project items).
- This is a built-in prop of shadcn SidebarMenuButton -- shows tooltip only when sidebar is in collapsed (icon) mode.

**Sidebar props:**
- `<Sidebar collapsible="icon">` -- enables the icon-only collapse mode per user decision (NOT expand-on-hover).

**Replace the entire dashboard layout** to use the sidebar instead of the top header nav.

The current layout has:
- Auth check (isPending -> loading, !session -> null, orgFix effect)
- `<header>` with nav links, notification bell, user email, logout button
- `<main>` with children

**New structure:**

Keep the EXACT same auth logic (isPending check, session redirect, orgFix useEffect, orgFixAttempted ref). These must remain BEFORE the SidebarProvider render. The pattern is:

```tsx
if (isPending) return loading spinner
if (!session) return null

return (
  <SidebarProvider defaultOpen={true}>
    <AppSidebar />
    <SidebarInset>
      <header className="flex h-14 items-center gap-2 border-b px-6">
        <SidebarTrigger />
        <div className="flex-1" />
        <NotificationBell />
      </header>
      <main className="flex-1 p-6">{children}</main>
    </SidebarInset>
  </SidebarProvider>
)
```

**Key changes from current layout:**
1. REMOVE the entire `<header>` block with nav links, user email, and logout button -- these are now in the sidebar.
2. REMOVE the nav `<Link>` elements for Projects, Metrics, Settings -- replaced by sidebar nav items.
3. REMOVE the user email `<span>` and logout `<Button>` from the header -- now in sidebar footer.
4. KEEP the `<NotificationBell />` component but move it to the new thin header inside SidebarInset.
5. ADD `<SidebarTrigger />` as the first element in the new header -- this is the hamburger/collapse button.
6. ADD `<AppSidebar />` import from `@/components/layout/app-sidebar`.
7. ADD `SidebarProvider`, `SidebarInset`, `SidebarTrigger` imports from `@/components/ui/sidebar`.

**Auth placement is critical:** SidebarProvider must be INSIDE the auth check (after `if (!session) return null`). Do NOT wrap SidebarProvider around the auth check or it will flash sidebar on unauthenticated routes.

**Session prop threading:** The AppSidebar needs the session for user email and logout. Two options:
- Option A: Pass session as prop from layout to AppSidebar.
- Option B: AppSidebar calls `useSession()` internally.
Use Option B (AppSidebar calls useSession internally) -- cleaner, the session is already cached by the auth client, and it avoids prop drilling. The AppSidebar already imports from auth-client per Task 1.

**Remove old imports** that are no longer used in layout.tsx:
- Remove `Link` from next/link (no more nav links in layout)
- Remove `usePathname` (no more active detection in layout)
- Remove `Button` from ui/button (no more logout button in layout)
- Remove `signOut` from auth-client (logout moved to sidebar)
- Remove the `handleLogout` function

**Keep these imports:**
- `useEffect`, `useRef` from react (for orgFix)
- `useRouter` from next/navigation (for auth redirect)
- `useSession`, `organization` from auth-client (for auth check and orgFix)
- `NotificationBell` from notifications
