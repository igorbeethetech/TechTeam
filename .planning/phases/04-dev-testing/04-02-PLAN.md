**Prompt builder -- `buildTestingPrompt(demand, project)`:**

The prompt must include:
1. System context: "You are a senior code reviewer and QA engineer. Review the code changes on the current branch against the original plan and requirements. Check code quality, correctness, and adherence to the plan. Run the project's test suite if tests exist. Produce a structured review report. IMPORTANT: Do NOT modify any code files. Your job is to READ and ANALYZE only. You may run tests using Bash (e.g., npm test, npx jest) but do NOT write or edit files."
2. Project context: name, tech stack, repo URL
3. Original requirements: `JSON.stringify(demand.requirements, null, 2)`
4. Original plan: `JSON.stringify(demand.plan, null, 2)`
5. Instructions for producing the verdict:
   - Run `git diff origin/{defaultBranch}...HEAD --stat` to see what files changed
   - Run `git diff origin/{defaultBranch}...HEAD` to see the full diff
   - Check if a test script exists in package.json; if so, run it
   - Compare implementation against each task in the plan
   - Compare implementation against each functional requirement
   - Rate each code quality issue by severity (critical, major, minor, suggestion)
   - Set verdict to "approved" only if: no critical issues, tests pass (or no tests exist), implementation addresses the plan tasks
   - Set verdict to "rejected" with clear rejectionReasons if any critical/major issues found or tests fail

Add `development` phase handler after the existing `planning` handler in the if/else chain. The full flow is:

```
development phase handler:
  1. Fetch demand (with plan, branchName, rejectionCount, testingFeedback) and project
  2. Validate repoPath exists and is a git repo (using validateGitRepo from git.ts). If not valid, throw error: "Project repoPath does not exist or is not a git repository"
  3. Branch management:
     - If demand.branchName is null (first run): create isolated branch using createIsolatedBranch(repoPath, branchName, defaultBranch). Generate branchName: `demand/${demand.id}-${slug}` where slug = title.toLowerCase().replace(/[^a-z0-9]+/g, '-').slice(0, 50). Update demand.branchName.
     - If demand.branchName exists (rejection re-entry): checkout existing branch: `createGitClient(repoPath).checkout(demand.branchName)`
  4. Invoke development agent: runDevelopmentAgent({ demandId, tenantId, projectId, timeout, rejectionFeedback: demand.testingFeedback ?? undefined })
  5. After agent completes, commit and push: commitAndPush(repoPath, demand.branchName, commitMessage). The commitMessage: use agentResult.output?.commitMessage if available, else `feat(demand/${demand.id}): ${demand.title}`
  6. PR management:
     - If demand.prUrl is null (first run): create PR using createPullRequest({ repoUrl: project.repoUrl, title: `[Demand ${demand.id}] ${demand.title}`, body: buildPrBody(demand, agentResult.output), head: demand.branchName, base: project.defaultBranch }). Update demand.prUrl.
     - If demand.prUrl exists (re-entry): skip PR creation (the push updates the existing PR automatically)
  7. Advance to testing: update demand { stage: "testing", agentStatus: "queued" }, enqueue testing job
  8. Error handling: on timeout/error, call resetWorkingDir(repoPath) to clean up dirty files, then re-throw
```

Add `testing` phase handler:

```
testing phase handler:
  1. Fetch demand (with plan, requirements, branchName, rejectionCount) and project
  2. Checkout demand branch: createGitClient(repoPath).checkout(demand.branchName)
  3. Invoke testing agent: runTestingAgent({ demandId, tenantId, projectId, timeout })
  4. If approved (testResult.approved === true):
     - Update demand { stage: "merge", agentStatus: null }
  5. If rejected (testResult.approved === false):
     - Increment rejectionCount
     - Check if rejectionCount >= 3 (MAX_REJECTION_CYCLES constant = 3):
       - If >= 3: Update demand { agentStatus: "paused", rejectionCount, testingFeedback: testResult.output }. Log "Demand paused after 3 rejection cycles".
       - If < 3: Update demand { stage: "development", agentStatus: "queued", rejectionCount, testingFeedback: testResult.output }. Enqueue development job.
  6. Error handling: on timeout/error, re-throw (no git cleanup needed since testing agent is read-only)
```

Define constant: `const MAX_REJECTION_CYCLES = 3`
