Update `apps/web/next.config.ts` to enable standalone output and monorepo file tracing:

```typescript
import path from "node:path"
import type { NextConfig } from "next"

const nextConfig: NextConfig = {
  output: "standalone",
  // CRITICAL for monorepo: trace dependencies from monorepo root, not just apps/web
  // Without this, workspace packages like @techteam/shared won't be included in standalone output
  outputFileTracingRoot: path.join(import.meta.dirname, "../../"),
  transpilePackages: ["@techteam/shared"],
}

export default nextConfig
```

Key points:
- `output: "standalone"` makes Next.js produce a self-contained server.js in `.next/standalone/`
- `outputFileTracingRoot` must point to monorepo root (`../../` from apps/web/) so the file tracer finds @techteam/shared
- `import.meta.dirname` is the ESM equivalent of `__dirname` (available in Node 22+, which is the project requirement)
- Keep `transpilePackages` for @techteam/shared (still needed for TypeScript compilation during build)

  <action>
Create `apps/web/Dockerfile` with the turbo prune three-stage pattern optimized for Next.js standalone:

**Stage 1 (pruner):** Use `node:22-alpine` base. Enable corepack, prepare pnpm@10.28.2. Add libc6-compat. Install turbo globally. COPY entire monorepo. Run `turbo prune @techteam/web --docker`.

**Stage 2 (builder):** Use same base. WORKDIR /app. Copy pruned json, lockfile, workspace yaml from pruner. Run `pnpm install --frozen-lockfile` (fall back to `pnpm install` if frozen fails). Copy full source from pruner. Copy turbo.json. Declare `ARG NEXT_PUBLIC_API_URL` for build-time Next.js public env var (NEXT_PUBLIC_* vars are baked at build time by Next.js -- they cannot be set at runtime). Set `ENV NEXT_PUBLIC_API_URL=${NEXT_PUBLIC_API_URL}`. Run `pnpm turbo build --filter=@techteam/web...`.

**Stage 3 (runner):** Use `node:22-alpine`. WORKDIR /app. Create nodejs group (gid 1001) and nextjs user (uid 1001). Switch to USER nextjs. Copy from builder with proper ownership:
- `/app/apps/web/.next/standalone` -> `./` (the entire standalone output to /app)
- `/app/apps/web/.next/static` -> `./apps/web/.next/static` (static assets)
- NOTE: No public/ directory exists in this project, so skip that COPY

Set `ENV HOSTNAME="0.0.0.0"` and `ENV PORT=3009`. EXPOSE 3009. CMD `["node", "apps/web/server.js"]`.

IMPORTANT:
- The `NEXT_PUBLIC_API_URL` build arg is essential. Without it, the frontend won't know where the API lives. In docker-compose, this is passed via `build.args`.
- The standalone output already includes all necessary node_modules files via Next.js file tracing -- no separate node_modules copy needed.
- There is no `public/` directory in this project, so do NOT include a COPY for public assets (it would fail the build).
