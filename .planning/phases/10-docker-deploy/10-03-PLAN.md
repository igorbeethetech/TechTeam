**2. Create `docker-compose.prod.yml`** at monorepo root with these services:

**postgres** service:
- image: postgres:16-alpine
- ports: "${POSTGRES_PORT:-5432}:5432" (note: dev compose uses 5433 externally, prod uses standard 5432)
- environment: POSTGRES_DB, POSTGRES_USER, POSTGRES_PASSWORD from env vars with defaults (techteam, techteam, no default for password -- it's required)
- volume: pgdata:/var/lib/postgresql/data
- healthcheck: pg_isready -U ${POSTGRES_USER:-techteam}, interval 5s, timeout 5s, retries 5
- restart: unless-stopped

**redis** service:
- image: redis:7-alpine
- ports: "${REDIS_PORT:-6379}:6379"
- command: redis-server --appendonly yes --maxmemory-policy noeviction
- volume: redisdata:/data
- healthcheck: redis-cli ping, interval 5s, timeout 5s, retries 5
- restart: unless-stopped

**api** service:
- build: context=., dockerfile=apps/api/Dockerfile
- ports: "${API_PORT:-3010}:3010"
- env_file: .env
- environment: DATABASE_URL constructed from postgres service vars (postgresql://${POSTGRES_USER:-techteam}:${POSTGRES_PASSWORD}@postgres:5432/${POSTGRES_DB:-techteam}), REDIS_URL=redis://redis:6379
- depends_on: postgres (service_healthy), redis (service_healthy)
- restart: unless-stopped

**web** service:
- build: context=., dockerfile=apps/web/Dockerfile, args: NEXT_PUBLIC_API_URL=${NEXT_PUBLIC_API_URL:-http://localhost:3010}
- ports: "${WEB_PORT:-3009}:3009"
- depends_on: api (no health condition needed, just startup order)
- restart: unless-stopped

**worker** service:
- build: context=., dockerfile=apps/api/Dockerfile (same image as API!)
- Use `entrypoint: []` to clear the migration entrypoint from the Dockerfile, then `command: ["node", "apps/api/dist/worker.js"]`. This ensures the worker does NOT run migrations -- only the API container does.
- env_file: .env
- environment: DATABASE_URL (same as api), REDIS_URL=redis://redis:6379
- volumes: repos:/app/repos (named volume for cloned repositories)
- depends_on: postgres (service_healthy), redis (service_healthy)
- restart: unless-stopped

**volumes**: pgdata, redisdata, repos (all named)

IMPORTANT design decisions:
- Worker uses the SAME Dockerfile as API (saves build time) but overrides the entrypoint to skip migrations. Only the API container runs migrations.
- DATABASE_URL is constructed in the environment section to use Docker's internal networking (hostname "postgres", not "localhost").
- REDIS_URL is hardcoded to docker internal redis hostname.
- The web service passes NEXT_PUBLIC_API_URL as a build arg (baked at build time by Next.js).
- repos volume is ONLY mounted on worker (it's the only service that does git operations).
