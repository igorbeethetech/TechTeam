  <action>
Create `scripts/docker-entrypoint-api.sh` (at monorepo root) with executable content:

```bash
#!/bin/sh
set -e

echo "[entrypoint] Running database migrations..."
cd /app/packages/database
npx prisma migrate deploy
echo "[entrypoint] Migrations complete."

echo "[entrypoint] Starting API server..."
cd /app
exec node apps/api/dist/server.js
```

Create `apps/api/Dockerfile` with the turbo prune three-stage pattern:

**Stage 1 (pruner):** Use `node:22-alpine` base. Enable corepack, prepare pnpm@10.28.2. Add libc6-compat. Install turbo globally (`pnpm add -g turbo@^2`). COPY entire monorepo. Run `turbo prune @techteam/api --docker`.

**Stage 2 (builder):** Use same base. WORKDIR /app. Copy pruned json files from pruner (`/app/out/json/`), pruned lockfile (`/app/out/pnpm-lock.yaml`), pruned workspace yaml (`/app/out/pnpm-workspace.yaml`). Run `pnpm install --frozen-lockfile`. If frozen-lockfile fails (turbo prune + pnpm 10 compatibility), fall back to `pnpm install`. Copy full source from pruner (`/app/out/full/`). Copy `turbo.json`. Run `pnpm turbo build --filter=@techteam/api...` (the `...` includes dependencies like @techteam/database and @techteam/shared).

**Stage 3 (runner):** Use `node:22-alpine`. Install git and openssl via apk (`RUN apk add --no-cache git openssl`). Create nodejs group (gid 1001) and appuser user (uid 1001). COPY entire built app from builder (`--chown=appuser:nodejs`). COPY entrypoint script from builder and chmod +x. USER appuser. EXPOSE 3010. ENTRYPOINT pointing to the entrypoint script.

IMPORTANT considerations:
- The database and shared packages export raw TypeScript (no build step). The `turbo build --filter=@techteam/api...` will run `prisma generate` for the database package (via the dependency chain) and `tsc` for the API.
- The runner stage copies the full /app because the API at runtime needs: dist/ (compiled JS), node_modules/, packages/database/generated/ (Prisma client), and packages/database/prisma/ (for migrate deploy).
- git is installed in runner because the worker container (same image, different CMD) needs it for simple-git operations.
- Do NOT add a .dockerignore yet -- that goes in Plan 03 with the compose file.
  </action>
  <verify>
Verify files exist: `ls apps/api/Dockerfile scripts/docker-entrypoint-api.sh`. Verify the entrypoint script references `prisma migrate deploy` (not `prisma migrate dev`). Verify the Dockerfile has all three stages (pruner, builder, runner). Verify ENTRYPOINT points to the entrypoint script.
  </verify>
