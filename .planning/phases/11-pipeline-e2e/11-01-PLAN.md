<objective>
Fix the git push authentication blocker that prevents Development and Merge phases from pushing to GitHub. Currently, `commitAndPush()` and `merge.worker.ts` push via `simple-git` which has no credentials configured. The tenant's GitHub token must be injected into the git remote URL before every push operation, and restored after to avoid token leakage in logs.

Also fix the merge worker's `closePullRequest` call which currently uses the default `getOctokit()` (env-only fallback) instead of passing the tenant's GitHub token.

Purpose: Without this fix, the Development phase fails at `git push` and the Merge phase fails at both `git push` and PR close. These are the only code blockers identified in the Phase 11 research -- all other pipeline code exists and should work.

Output: Three modified files with git authentication properly wired through the pipeline.
</objective>
