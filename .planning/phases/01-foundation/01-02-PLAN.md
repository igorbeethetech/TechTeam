  <action>
    **Step 1: Install Better Auth in apps/api.**
    Run `pnpm --filter @techteam/api add better-auth`

    **Step 2: Update Prisma schema with Better Auth tables.**
    Run `npx @better-auth/cli generate` from the packages/database directory to discover what tables Better Auth needs. Then manually add these tables to `packages/database/prisma/schema.prisma` alongside the existing Project model.

    Better Auth with organization plugin typically needs these models (use the CLI output as ground truth, but expect):
    - `user`: id, name, email (unique), emailVerified, image, createdAt, updatedAt
    - `session`: id, expiresAt, token (unique), createdAt, updatedAt, ipAddress, userAgent, userId (FK to user)
    - `account`: id, accountId, providerId, userId (FK to user), accessToken, refreshToken, idToken, accessTokenExpiresAt, refreshTokenExpiresAt, scope, password, createdAt, updatedAt
    - `verification`: id, identifier, value, expiresAt, createdAt, updatedAt
    - `organization`: id, name, slug (unique), logo, createdAt, metadata
    - `member`: id, organizationId (FK to organization), userId (FK to user), role, createdAt
    - `invitation`: id, organizationId (FK to organization), email, role, status, expiresAt, inviterId (FK to user)

    IMPORTANT: Use lowercase model names as Better Auth expects them. Map them using `@@map` if needed, or just use lowercase names directly.

    After updating schema, run:
    1. `pnpm --filter @techteam/database db:generate` (regenerate Prisma client)
    2. `pnpm --filter @techteam/database db:migrate` with migration name "add-auth-tables"

    **Step 3: Create Better Auth server instance.**
    Create `apps/api/src/auth.ts` using RESEARCH.md Pattern 3. Key config:
    - `database`: prismaAdapter(prisma, { provider: "postgresql" })
    - `emailAndPassword.enabled`: true
    - `session.expiresIn`: 60 * 60 * 24 * 7 (7 days)
    - `session.updateAge`: 60 * 60 * 24 (refresh daily)
    - `session.cookieCache.enabled`: true, maxAge: 5 * 60 (5-min cookie cache)
    - `trustedOrigins`: [process.env.WEB_URL || "http://localhost:3000"]
    - `basePath`: "/api/auth"
    - `plugins`: [organization()] -- this adds activeOrganizationId to sessions
    - Import prisma from @techteam/database

    **Step 4: Create auth catch-all route.**
    Create `apps/api/src/routes/auth.ts` using RESEARCH.md Pattern 3 (Fastify integration). The route handles ALL requests to `/api/auth/*` by converting Fastify request to Web Request, passing to `auth.handler()`, and forwarding the response back.

    CRITICAL: When constructing the Web Request:
    - Set correct Content-Type header from the original request
    - Handle request body correctly (Fastify may pre-parse JSON, so use `JSON.stringify(request.body)` for non-GET requests)
    - Forward ALL response headers from Better Auth back to the Fastify reply (especially Set-Cookie)
    - Handle cases where response body might not be JSON (use `.text()` as fallback)

    **Step 5: Create auth preHandler plugin.**
    Create `apps/api/src/plugins/auth.ts` as a Fastify plugin using `fastify-plugin`:
    - Install fastify-plugin: `pnpm --filter @techteam/api add fastify-plugin`
    - Decorate request with `session` and `user` properties
    - In preHandler hook: Extract session token from request headers/cookies, validate via Better Auth `auth.api.getSession()`, attach to request
    - If no valid session, return 401

    This plugin will be registered on protected route scopes (not globally -- auth routes must remain public).

    **Step 6: Update server.ts to register auth routes.**
    Modify `apps/api/src/server.ts` (from Plan 01):
    - Import and register authRoutes (public, no auth middleware)
    - Keep the /health endpoint
    - Do NOT add protected routes yet -- those come in Plan 03
    - The server structure should follow RESEARCH.md "Fastify Server Setup" pattern but only the auth-relevant parts
  </action>
  <verify>

    6. Test login: `curl -X POST http://localhost:3001/api/auth/sign-in/email -H "Content-Type: application/json" -d '{"email":"test@example.com","password":"password123"}' -c cookies.txt` -- should return session and set cookies
    7. Test session: `curl http://localhost:3001/api/auth/get-session -b cookies.txt` -- should return session object
  </verify>

    import { createAuthClient } from "better-auth/react"
    import { organizationClient } from "better-auth/client/plugins"
    export const authClient = createAuthClient({
      baseURL: process.env.NEXT_PUBLIC_API_URL || "http://localhost:3001",
      plugins: [organizationClient()],

    export const { signIn, signUp, signOut, useSession, organization } = authClient
