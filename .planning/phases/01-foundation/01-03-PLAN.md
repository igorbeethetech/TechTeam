    **Step 1: Create forTenant() Prisma Client Extension.**
    Create `packages/database/src/tenant.ts` using the EXACT pattern from RESEARCH.md Pattern 2. The function:
    - Takes a tenantId string parameter
    - Returns `prisma.$extends(...)` with a query component on `$allModels.$allOperations`
    - For TENANT_MODELS (currently just "Project" -- User is managed by Better Auth separately):
      - Read operations (findMany, findFirst, findUnique, count, aggregate): inject `tenantId` into `args.where`
      - Create: inject `tenantId` into `args.data`
      - CreateMany: inject `tenantId` into each item in `args.data` array
      - Update/UpdateMany/Delete/DeleteMany: inject `tenantId` into `args.where`
    - For non-tenant models: pass through to `query(args)` unchanged
    - Export `forTenant` function

    Update `packages/database/src/index.ts`: Add `export { forTenant } from "./tenant.js"`

    **Step 2: Create tenant Fastify plugin.**
    Create `apps/api/src/plugins/tenant.ts` using RESEARCH.md Pattern 2 (Fastify plugin section):
    - Use fastify-plugin (fp) wrapper
    - Decorate request with `prisma` property (initially null)
    - In preHandler hook:
      - Get tenantId from `request.session.session.activeOrganizationId` (set by Better Auth organization plugin -- the exact path depends on how auth plugin attaches session; use the auth preHandler from Plan 02 which already validated the session and attached it to request)
      - If no tenantId (no active organization): return 403 with `{ error: "No active organization. Please select or create an organization." }`
      - Set `request.prisma = forTenant(tenantId)` -- this gives the route handler a tenant-scoped Prisma client
    - Add TypeScript declaration merging to extend FastifyRequest with `prisma` and `session`/`user` properties

    **Step 3: Create project CRUD routes.**
    Create `apps/api/src/routes/projects.ts` as a Fastify plugin with prefix "/api/projects":

    **GET /** - List projects
    - Uses `request.prisma.project.findMany({ where: { status: "active" }, orderBy: { createdAt: "desc" } })`
    - tenantId filtering is automatic via the extension
    - Returns `{ projects: Project[] }`

    **POST /** - Create project
    - Validate body with `projectCreateSchema` from @techteam/shared (import Zod schema)
    - If validation fails, return 400 with Zod error messages
    - `request.prisma.project.create({ data: validatedData })`
    - tenantId is automatically injected by the extension
    - Returns 201 with `{ project: Project }`

    **GET /:id** - Get single project
    - `request.prisma.project.findUnique({ where: { id: params.id } })`
    - tenantId filtering is automatic -- if project belongs to different tenant, findUnique returns null
    - If null, return 404 with `{ error: "Project not found" }`
    - Returns `{ project: Project }`

    **PUT /:id** - Update project
    - Validate body with `projectUpdateSchema` from @techteam/shared
    - `request.prisma.project.update({ where: { id: params.id }, data: validatedData })`
    - tenantId is in the where clause automatically
    - If project not found (wrong tenant or doesn't exist), catch Prisma error and return 404
    - Returns `{ project: Project }`

    **PATCH /:id/archive** - Archive project
    - `request.prisma.project.update({ where: { id: params.id }, data: { status: "archived" } })`
    - Returns `{ project: Project }`

    **PATCH /:id/unarchive** - Unarchive project (useful for mistakes)
    - `request.prisma.project.update({ where: { id: params.id }, data: { status: "active" } })`
    - Returns `{ project: Project }`

    **Step 4: Update server.ts to wire protected routes.**
    Modify `apps/api/src/server.ts`:
    - Import auth plugin (from Plan 02) and tenant plugin
    - Create a protected scope that registers both auth and tenant plugins
    - Register projectRoutes within the protected scope
    - Structure follows RESEARCH.md "Fastify Server Setup" pattern:
      ```
      // Public routes
      register(authRoutes)

      // Protected routes (auth + tenant middleware)
      register(async (protectedApp) => {
        register(authPlugin)   // validates session
        register(tenantPlugin) // scopes prisma to tenant
        register(projectRoutes, { prefix: "/api/projects" })
      })
      ```
